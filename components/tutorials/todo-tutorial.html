Building a Todo App with Ayisha.js

Learn how to build a complete todo application step by step using Ayisha.js reactive framework.

Step 1: Getting Started with @model

Let's start by creating a simple input field that binds to our data using the @model directive.

Code:

<div class="todo-app-container">
  <div class="todo-header">
    <h2>My Todo App</h2>
    <p>Built with Ayisha.js</p>
  </div>
  <div class="todo-input-section">
    <input type="text" @model="newTodo" class="todo-input" placeholder="What needs to be done?">
  </div>
  <p>You typed: <span @text="newTodo"></span></p>
</div>

Step 2: Event Handling with @click

Now let's add a button to add todos to our list using the @click directive.

Code:

<div class="todo-app-container">
  <div class="todo-header">
    <h2>My Todo App</h2>
    <p>Built with Ayisha.js</p>
  </div>
  <div class="todo-input-section">
    <input type="text" @model="newTodo" class="todo-input" placeholder="What needs to be done?">
    <div class="todo-buttons">
      <button @click="todos.push(newTodo); newTodo = ''" class="todo-btn">Add Todo</button>
      <button @click="todos = []" class="todo-btn todo-btn-secondary">Clear All</button>
    </div>
  </div>
  <p>Total todos: <span @text="todos.length"></span></p>
</div>

Step 3: Displaying Lists with @for

Let's display our todos in a list using the @for directive to iterate over the array.

Code:

    </div>

Step 4: Conditional Rendering with @if

Let's add conditional rendering to show different messages based on whether we have todos or not.

Code:

<div class="todo-app-container">
  <div class="todo-header">
    <h2>My Todo App</h2>
    <p>Built with Ayisha.js</p>
  </div>
  <div class="todo-input-section">
    <input type="text" @model="newTodo" class="todo-input" placeholder="What needs to be done?">
    <div class="todo-buttons">
      <button @click="todos.push(newTodo); newTodo = ''" class="todo-btn">Add Todo</button>
      <button @click="todos = []" class="todo-btn todo-btn-secondary">Clear All</button>
    </div>
  </div>
  <div @if="todos.length === 0" class="empty-state">
    <p>No todos yet. Add one above!</p>
  </div>
  <ul @if="todos.length > 0" class="todo-list">
    <li @for="todo in todos" class="todo-item">
      <span @text="todo"></span>
    </li>
  </ul>
</div>

Step 6: Interactive Checkboxes

Let's add checkboxes to mark todos as completed using two-way data binding.

Code:

<div class="todo-app-container">
  <div class="todo-header">
    <h2>My Todo App</h2>
    <p>Built with Ayisha.js</p>
  </div>
  <div class="todo-input-section">
    <input type="text" @model="newTodo" class="todo-input" placeholder="What needs to be done?">
    <div class="todo-buttons">
      <button @click="todos.push({text: newTodo, completed: false}); newTodo = ''" class="todo-btn">Add Todo</button>
      <button @click="todos = []" class="todo-btn todo-btn-secondary">Clear All</button>
    </div>
  </div>
  <div class="todo-stats">
    <span @text="todos.filter(t => !t.completed).length"></span> active
    <span @text="todos.filter(t => t.completed).length"></span> completed
  </div>
  <ul @if="todos.length > 0" class="todo-list">
    <li @for="todo in todos" class="todo-item">
      <label class="todo-checkbox-label">
        <input type="checkbox" @model="todo.completed">
        <span @text="todo.text"></span>
      </label>
    </li>
  </ul>
</div>

Step 8: Filtering Lists with @show

Let's add filtering functionality to show all, active, or completed todos using the @show directive.

Code:

<div class="todo-app-container">
  <div class="todo-header">
    <h2>My Todo App</h2>
    <p>Built with Ayisha.js</p>
  </div>
  <div class="todo-input-section">
    <input type="text" @model="newTodo" class="todo-input" placeholder="What needs to be done?">
    <div class="todo-buttons">
      <button @click="todos.push({text: newTodo, completed: false}); newTodo = ''" class="todo-btn">Add Todo</button>
      <button @click="todos = []" class="todo-btn todo-btn-secondary">Clear All</button>
    </div>
  </div>
  <div class="todo-stats">
    <span @text="todos.filter(t => !t.completed).length"></span> active
    <span @text="todos.filter(t => t.completed).length"></span> completed
  </div>
  <div class="todo-filters">
    <button @click="filter = 'all'" class="todo-filter-btn" @style="{backgroundColor: filter === 'all' ? '#007bff' : '#f8f9fa', color: filter === 'all' ? 'white' : '#333'}">All</button>
    <button @click="filter = 'active'" class="todo-filter-btn" @style="{backgroundColor: filter === 'active' ? '#007bff' : '#f8f9fa', color: filter === 'active' ? 'white' : '#333'}">Active</button>
    <button @click="filter = 'completed'" class="todo-filter-btn" @style="{backgroundColor: filter === 'completed' ? '#007bff' : '#f8f9fa', color: filter === 'completed' ? 'white' : '#333'}">Completed</button>
  </div>
  <ul @if="todos.length > 0" class="todo-list">
    <li @for="todo in todos" @show="filter === 'all' || (filter === 'active' && !todo.completed) || (filter === 'completed' && todo.completed)" class="todo-item" @style="{opacity: todo.completed ? 0.6 : 1}">
      <label class="todo-checkbox-label">
        <input type="checkbox" @model="todo.completed">
        <span @text="todo.text" @style="{textDecoration: todo.completed ? 'line-through' : 'none'}"></span>
      </label>
    </li>
  </ul>
</div>

What You've Learned:

  • @model - Two-way data binding for inputs
  • @click - Event handling for user interactions
  • @for - Iterating over arrays to display lists
  • @if/@show - Conditional rendering and visibility
  • @text - Dynamic text content
  • @style - Dynamic styling based on state
  • State Management - Working with objects and arrays
  • Filtering - Showing/hiding items based on conditions
  • CRUD Operations - Create, Read, Update, Delete functionality

You now have a solid foundation in Ayisha.js and can build interactive web applications!
